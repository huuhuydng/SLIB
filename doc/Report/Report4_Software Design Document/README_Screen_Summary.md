# 3.1.4 Non-Screen Functions

| # | Feature | System Function | Description |
|---|---|---|---|
| 1 | Authentication Module | Session Validation | The system checks whether a valid login session already exists and redirects users to the correct entry flow. |
| 2 | Authentication Module | Token Expiration Handling | The system automatically detects expired tokens and forces users to re-authenticate when necessary. |
| 3 | Booking Seat Module | Automatic Booking Timeout | The system automatically cancels unconfirmed bookings when the confirmation countdown expires. |
| 4 | Booking Seat Module | Real-time Seat Update | The system synchronizes seat availability and seat status in real time through WebSocket updates. |
| 5 | Booking Seat Module | Seat Recommendation | The system generates AI-based seat suggestions for Student and Teacher users. |
| 6 | Library Access Module | Automatic Access Status Check | The system checks whether the user is currently checked in before deciding the QR scan flow. |
| 7 | System Configuration Module | Automatic Backup Schedule | The system can back up data automatically based on configured scheduling rules. |
| 8 | Chat & Support Module | Conversation Persistence | The system restores active chat sessions and saved chat state when the user reopens the mobile app. |
| 9 | Chat & Support Module | Chat Escalation | The system escalates an AI conversation to a Librarian when human support is requested or required. |
| 10 | Chat & Support Module | AI Suggested Response | The system provides AI-generated suggestions to support Librarian responses in chat workflows. |
| 11 | Notification Module | Real-time Notification Delivery | The system pushes and refreshes notifications based on new activities and status changes. |
| 12 | Feedback Module | Pending Feedback Reminder | The system detects completed sessions with pending feedback and prompts the user to submit feedback. |
| 13 | System Configuration Module | Kiosk Activation Validation | The system validates kiosk activation tokens before enabling kiosk routes and screens. |
| 14 | News & Announcement Module | Public Content Synchronization | The system loads and refreshes public news and new book content from cache first, then from server data. |
