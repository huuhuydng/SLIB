# Entity Descriptions

| # | Entity | Description |
|---|---|---|
| 1 | User | Central entity representing all system users, including Admins, Librarians, Students, and Teachers, for authentication and role management. |
| 2 | User Setting | Stores individual user preferences and application settings such as HCE, AI recommendation, reminders, theme, and language. |
| 3 | Student Profile | Stores student-specific profile information and statistics such as reputation score, study hours, and violation count. |
| 4 | Refresh Token | Manages authentication refresh tokens for secure login sessions and token rotation. |
| 5 | Area | Represents a top-level physical library area in the layout management model. |
| 6 | Zone | Represents a sub-area inside a library area where seats and facilities are organized. |
| 7 | Zone Amenity | Represents an amenity or attribute assigned to a specific zone. |
| 8 | Seat | Represents an individual seat with booking status, occupancy state, and optional NFC tag assignment. |
| 9 | Library Setting | Stores system-wide operational rules such as opening hours, booking rules, and library availability state. |
| 10 | HCE Device | Represents an HCE scan station used for access control and check-in or check-out operations. |
| 11 | Reservation | Represents a seat booking transaction created by a user for a defined time period. |
| 12 | Access Log | Stores check-in and check-out activity for library access and seat usage. |
| 13 | Activity Log | Stores user activity history and traceable system actions related to reservation or usage behavior. |
| 14 | Reputation Rule | Defines a reputation scoring rule used to reward or penalize user behavior. |
| 15 | Point Transaction | Represents a reputation score change applied to a user, including rule source and resulting balance. |
| 16 | Complaint | Represents a formal complaint submitted by a user to dispute a penalty or violation-related decision. |
| 17 | Feedback | Stores feedback submitted by users after service usage, including rating, content, and review state. |
| 18 | Seat Status Report | Represents a report about seat condition problems such as damage, maintenance issues, or physical defects. |
| 19 | Seat Violation Report | Represents a report about seat misuse, occupied seat violations, or rule-breaking behavior. |
| 20 | Notification | Stores in-app notifications sent to users about system, booking, report, or communication events. |
| 21 | News Category | Classifies news and announcement content into manageable categories. |
| 22 | News | Represents a news article or announcement published in the system. |
| 23 | New Book | Represents a newly introduced book entry shown in the content and discovery features. |
| 24 | Conversation | Represents a human support conversation between a student and a librarian. |
| 25 | Message | Represents an individual message exchanged in a support conversation. |
| 26 | Support Request | Represents a manual support request submitted by a student for librarian assistance. |
| 27 | Kiosk Config | Represents a kiosk device definition, activation state, and security token metadata. |
| 28 | Kiosk Image | Represents an image item displayed in the kiosk slideshow feature. |
| 29 | Library Map | Represents an uploaded visual map used for kiosk display or map-based navigation. |
| 30 | Zone Map | Represents an interactive zone layer positioned on top of a library map. |
| 31 | Kiosk QR Session | Represents a temporary kiosk QR interaction session used during access-related flows. |
| 32 | Kiosk Activation Code | Represents an activation code used to bind and activate kiosk devices securely. |
| 33 | AI Knowledge Store | Represents a knowledge repository used by AI-related features for retrieval and indexing. |
| 34 | AI Material | Represents a material set attached to a knowledge store. |
| 35 | AI Material Item | Represents an individual item, file, or text unit inside an AI material set. |
| 36 | Chat Session | Represents a user-owned AI chat session in the assistant subsystem. |
| 37 | Chat Message | Represents an individual AI chat message stored inside a chat session. |
| 38 | Knowledge Base | Represents structured internal knowledge used by AI services for response generation and retrieval. |
| 39 | Prompt Template | Represents a reusable AI prompt template for predefined system contexts. |
| 40 | Student Behavior | Represents analytics or AI-derived behavioral records linked to a student’s bookings, seats, or zones. |
| 41 | Backup Schedule | Represents the configuration for automatic database backup execution. |
| 42 | Backup History | Stores the execution history and result of backup operations. |
| 43 | System Log | Represents an audit or monitoring log entry generated by the system. |
| 44 | Layout Draft | Represents an unpublished library layout draft created during layout administration. |
| 45 | Layout History | Represents a published version or historical snapshot of the library layout. |

## Notes

- The entities listed here are aligned with the current relational schema and core business objects of the system.
- The descriptions are written at conceptual entity level for report readability, so some low-level technical tables are intentionally summarized into clearer business entities.
